# Frontier Firm Agent Starter Kits

A simple, browse-and-download hub for ready-to-use **Microsoft Copilot Studio / Microsoft 365 Copilot** agent templates.

Each "starter kit" is a self-contained ZIP that gives you everything you need to understand, build, set up, and evaluate a production-ready agent — no guesswork, no starting from a blank page.

---

## What is this?

Building a good Copilot agent from scratch is hard. You have to figure out the use case, gather the right knowledge files, configure it correctly, and then prove it actually works.

**Frontier Firm Agent Starter Kits** packages all of that into downloadable kits. Pick an agent, download the ZIP, and you get:

- 📊 **A pitch deck** — a PowerPoint explaining the agent and the business value it delivers.
- 📁 **Example knowledge files** — sample content to ground the agent so you can see it working immediately.
- 📄 **A setup & evaluation guide** — a Word document walking you through how to stand the agent up and how to measure whether it's performing well.

> Kit contents vary slightly by agent, but every kit is designed to take you from "interesting idea" to "working, evaluated agent" as fast as possible.

---

## Who is this for?

- **Internal teams** standardizing how they build and roll out Copilot agents.
- **External partners, customers, and the community** who want a proven, opinionated starting point instead of a blank canvas.

If you work with Copilot Studio or Microsoft 365 Copilot and want a head start, this is for you.

## How to download a kit

This project is designed to be **UI-first and dead simple**:

1. Open the hub (the landing page).
2. Browse the available agent kits.
3. Click the kit you want.
4. You're taken to GitHub to download the ZIP.
5. Unzip and follow the included setup & evaluation guide.

That's the whole flow. No accounts, no forms, no friction.

---

## How it's hosted

- The **landing page** is a lightweight static site hosted on GitHub (e.g. GitHub Pages).
- The **ZIP downloads** are published as **GitHub Release assets**, which keeps the repo lightweight, gives each kit a clean versioned download link, and lets us track downloads over time.
- The site is **static and read-only** — nothing is collected from visitors, and there's no backend.

## Design & branding

This is a **Microsoft** site, and the UI takes its visual cues directly from Microsoft's modern dark theme:

- **Dark background** — deep near-black with subtle blue/purple ambient glow.
- **Gradient headline text** — key phrases flow from **blue → purple → pink**.
- **Colorful kit icons** — each agent card uses a bright, distinct emoji/icon accent (greens, oranges, pinks, blues) against the dark cards, so the catalog stays vibrant and scannable.
- **Clean, minimal layout** — generous spacing, centered hero, and a simple grid of cards. UI-first, nothing extra.

The goal is a polished, on-brand Microsoft feel that's instantly familiar.

## Project goals

- **Straightforward UI only** — see the options, click one, download. Nothing more.
- **Self-contained kits** — everything needed to understand, build, and evaluate an agent in one ZIP.
- **Easy to grow** — adding a new kit should be as simple as publishing a new release and adding a card.
